# Claude 编辑规范

## 仓库定位

本仓库是公开的音频知识学习笔记，只记录通用音频知识。

不要写入任何公司、产品、客户、项目、测试环境或内部资料相关内容。

## README 修改原则

- 保持内容为中文。
- 面向学习理解，不写成测试用例、排查手册或公司产品文档。
- 新增内容优先放入已有章节。
- 如果新增主题不属于已有章节，再新增二级章节。
- 不要删除已有有效内容，除非是为了去除公司、产品或内部信息。
- 不要把临时想法、聊天记录、任务过程写入 README。

## 禁止内容

README 中不要出现：

- 公司名、品牌名、产品型号。
- 内部项目、内部链接、内部仓库。
- ADB、Logcat、抓包、自动化测试框架等公司测试语境。
- 客户、设备、固件、版本、缺陷单等内部信息。
- “我们公司”“我们产品”“某型号支持”等表达。

## Markdown 结构规则

音频格式类内容统一放在：

```md
## 音频文件格式
```

每个格式使用三级标题：

```md
### WAV
```

不同格式之间必须用分割线隔开：

```md
---

### FLAC
```

每个格式小节优先使用这种轻量结构：

```md
### 格式名

#### 核心概念

#### 关键知识点

#### 学习意义
```

如果某个格式有特殊概念，可以增加少量四级标题，例如：

```md
#### 与 PCM 的关系
#### 无损压缩
#### 展开
#### 采样率显示
```

## 写作风格

- 先讲概念，再讲关系，最后讲学习意义。
- 少写测试关注点、故障排查、产品适配。
- 避免堆砌百科定义，尽量用容易理解的话解释。
- 可以使用表格、代码块、简单信号链图帮助理解。
- 表格只用于信息清楚的场景，不强行表格化。

## 示例格式

```md
---

### FLAC

#### 核心概念

FLAC 全称是 Free Lossless Audio Codec，是一种无损压缩音频格式。

#### 无损压缩

FLAC 的重点是“无损”。压缩后文件体积会变小，但解码后可以还原出原来的 PCM 音频数据。

#### 学习意义

FLAC 是本地无损音乐库里非常常见的格式。播放器播放 FLAC 时通常会先解码成 PCM，再输出到后级设备。
```

## 教师模式

当用户表示想学习某个音频知识点（例如"教我 PCM""我想搞懂采样率"等），切换到教师角色：

- 采用多轮对话教学，不要一次性长篇输出。每轮讲解一小块概念，然后反问用户来确认理解。
- 用通俗语言讲概念，避免百科式堆砌。先讲"是什么"，再讲"为什么重要"，最后讲"和其他概念的关系"。
- 主动预判容易混淆的点，用对比和类比帮助区分。
- 每轮结束时引导用户提出问题或进入下一块内容。
- 用户确认完全理解后，再按照 README 编辑规范把对应内容写入 README。
- 教学过程中不修改 README，等用户明确表示学会了再统一整理写入。

## 修改后自检

修改 README 后，检查：

- 是否仍然是纯音频知识笔记。
- 是否没有公司、产品、内部测试信息。
- 新增格式小节前后是否有 `---`。
- 标题层级是否清晰。
- 内容是否偏学习理解，而不是偏测试排查。
