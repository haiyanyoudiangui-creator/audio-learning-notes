### PCM

#### 核心概念

PCM 全称是 Pulse Code Modulation（脉冲编码调制），是把连续的声波变成数字数据的编码方式。

```text
PCM = 采样（Sampling）+ 量化（Quantization）
```

- **采样**：每隔固定时间，在波形上取一个点。相当于给波形拍照，一秒拍很多次。
- **量化**：每个点的高度（振幅），用一个数字记录下来。这个数字的精度就是位深。

类比：在方格纸上描一条曲线。横向格子密度对应采样率——格子越密，描出的曲线越接近原形。纵向刻度精度对应位深——刻度越细，记录的高度越精确。描完后不再需要原曲线，只看这些格点就能把波形大致还原。

#### 采样率

采样率是每秒钟采样的次数，单位 Hz。

**奈奎斯特定理**：要完整还原一个频率的信号，采样率至少要是它的两倍。

人耳能听到的最高频率约 20kHz，所以 CD 标准选了 44.1kHz——刚好是 20kHz 的两倍再加一点安全余量。

常见采样率：

| 采样率 | 能还原的最高频率 | 典型用途 |
| --- | --- | --- |
| 44.1kHz | ~22kHz | CD |
| 48kHz | ~24kHz | 视频、电影 |
| 96kHz | ~48kHz | 高解析音频、录音 |
| 192kHz | ~96kHz | 母带级录音 |

高采样率（96kHz / 192kHz）的主要价值不是记录超声波给人听，而是为了给抗混叠滤波器提供更宽的过渡带，让滤波器设计更容易、失真更小。同时在录音混音阶段提供处理余量和更低延迟。

#### 抗混叠滤波器

采样前必须用低通滤波器把超过采样率一半的频率砍掉，否则这些超高频会伪装成低频混入可听频段——这叫**频率混叠（aliasing）**。这个滤波器叫抗混叠滤波器。

滤波器的频段划分：

| 区域 | 含义 | 44.1kHz 采样时的值 |
| --- | --- | --- |
| 保留区 | 信号完全放行，是要保护的音频 | 0Hz ~ 20kHz（人耳上限） |
| 阻断区 | 信号全部拦截 | 22.05kHz 以上（44.1kHz ÷ 2） |
| 过渡带 | 从放行过渡到拦截的缓冲区 | 20kHz ~ 22.05kHz |

过渡带宽度 = 阻断区起点 − 保留区终点。44.1kHz 采样时过渡带只有约 2kHz，滤波器必须在极窄范围内剧烈衰减，设计难度大、容易引入失真。96kHz 采样时阻断区起点在 48kHz，过渡带宽度达到 28kHz，滤波器可以从容缓降，可听频段的保真度更好。

```text
44.1kHz 采样：保留区 [0~20kHz]  过渡带 2kHz  阻断区 [22.05kHz~]
96kHz 采样：  保留区 [0~20kHz]  过渡带 28kHz 阻断区 [48kHz~]
```

#### 位深

位深是每个采样点的振幅用多少 bit 记录。bit 数越多，刻度越密，记录越精确。

| 位深 | 可用刻度数 | 动态范围 |
| --- | --- | --- |
| 8bit | 256 | ~48dB |
| 16bit | 65,536 | ~96dB |
| 24bit | 16,777,216 | ~144dB |

每多 1bit，可表示的值翻倍。在 dB 的世界里，翻倍约等于 6dB，所以每多 1bit 约增加 6dB 动态范围。

```text
动态范围(dB) ≈ 6.02 × n + 1.76    （n = 位深）
简记：每 bit ≈ 6dB
```

16bit 约 96dB 的动态范围已经覆盖了从安静房间到喷气式飞机起飞的声音跨度。24bit 约 144dB，主要优势在于录音和混音阶段——极大的动态余量让录音师不用在"怕削顶"和"怕吃底噪"之间反复权衡，后期处理空间充足。

#### 码率

码率是每秒产生的音频数据量，由采样率和位深共同决定。

```text
码率 = 采样率 × 位深 × 声道数
```

以 CD 规格为例：

```text
44.1kHz × 16bit × 2声道
= 44100 × 16 × 2
= 1,411,200 bps
≈ 1411 kbps
```

文件大小由码率和时长决定：

```text
文件大小 ≈ 码率 × 时长
CD 规格一分钟 ≈ 1411kbps × 60s ≈ 10.6MB
```

不同场景的码率对比：

| 场景 | 计算 | 码率 |
| --- | --- | --- |
| CD 音质 | 44.1k × 16 × 2 | ~1411 kbps |
| 高解析立体声 | 96k × 24 × 2 | ~4608 kbps |
| 高质量有损流媒体（MP3） | — | ~320 kbps |

#### 学习意义

PCM 是数字音频的基石。理解 PCM 之后，WAV（装 PCM 的容器）、FLAC（无损压缩 PCM）、MP3 / AAC（有损压缩 PCM）以及 DSD（另一种不同的编码方式）都建立在它的概念之上。

```text
采样率 → 每秒取多少个点 → 决定能记录的最高频率
位深   → 每个点记多准   → 决定动态范围
码率   → 上面两个的乘积 → 决定每秒数据量和文件大小
```

### DSD 编码方式

#### 核心概念

DSD 全称是 Direct Stream Digital，是一种与 PCM 不同的数字音频编码方式。

PCM 用"采样 + 量化"的方式记录音频——在每个采样点上记录一个多比特的数值（如 16bit、24bit）。DSD 则完全不同：它用极高速率的 1-bit 位流来表示音频，每一位只表示信号"上升"还是"下降"。

```text
PCM：多比特量化 — 每个采样点记录波形的"位置"
DSD：1-bit 位流 — 记录波形的"变化方向"
```

两者不是文件格式上的区别，而是**编码方式**上的根本差异。

#### 与 PCM 的关系 / 对比

| 特性 | PCM | DSD |
| --- | --- | --- |
| 编码方式 | 脉冲编码调制（PCM） | 脉冲密度调制（PDM） |
| 量化方式 | 多比特（16bit / 24bit / 32bit） | 1-bit |
| 记录内容 | 每个采样点的振幅值 | 振幅的增减方向 |
| 采样率 | 44.1kHz、48kHz、96kHz、192kHz 等 | DSD64（约 2.8MHz）、DSD128（约 5.6MHz）等 |
| 常见载体 | WAV、FLAC、MP3、AAC 等 | DSF、DFF、SACD ISO |

简单来说：PCM 和 DSD 是两条路，从编码方式、到文件格式、再到播放和解码链路都不一样。WAV 是 PCM 体系里的容器，DSF 是 DSD 体系里的容器，它们才是对等概念。

#### DSD 规格

DSD 常见规格用"DSD + 数字"表示，数字代表采样率是 CD 采样率（44.1kHz）的多少倍：

| 规格 | 采样率 | 说明 |
| --- | --- | --- |
| DSD64 | 约 2.8MHz（44.1kHz × 64） | 最常见的 DSD 规格，SACD 的标准 |
| DSD128 | 约 5.6MHz（44.1kHz × 128） | 比 DSD64 高一倍 |
| DSD256 | 约 11.2MHz（44.1kHz × 256） | 更高规格 |
| DSD512 | 约 22.6MHz（44.1kHz × 512） | 极高规格，文件非常庞大 |

#### 学习意义

理解 DSD 是一种编码方式而不是文件格式，有助于理清整条音频链路：

```text
DSD 编码 → DSF / DFF 文件 → 播放器 → 输出方式（DoP / D2P / Native）→ DAC
```

后续在文件格式章节会讲 DSF / DFF（DSD 的容器），在输出与解码章节会讲 DSD 的三种输出方式。

### WAV

#### 核心概念

WAV 全称通常写作 Waveform Audio File Format，是一种常见的音频容器格式。

它本身更像一个"文件壳"，里面最常见装的是 PCM 音频数据。

```text
WAV 通常 = PCM 原始音频数据 + 文件头信息
```

#### 与 PCM 的关系

PCM 是音频数据的编码方式，WAV 是文件格式。

```text
PCM = 音频数据怎么存
WAV = 把 PCM 数据装进一个文件里
```

所以常见的 `16bit / 44.1kHz WAV`、`24bit / 96kHz WAV`，实际描述的是 WAV 文件里面 PCM 数据的规格。

#### 文件信息

一个 WAV 文件通常会记录：

| 信息 | 说明 |
| --- | --- |
| 采样率 | 例如 44.1kHz、48kHz、96kHz、192kHz |
| 位深 | 例如 16bit、24bit、32bit |
| 声道数 | 单声道、双声道、多声道 |
| 编码格式 | 最常见是 PCM，也可能是 ADPCM、浮点 PCM 等 |
| 音频数据 | 真正的声音采样数据 |

#### 优点与缺点

| 类型 | 内容 |
| --- | --- |
| 优点 | 结构简单、兼容性强、通常无压缩、适合分析音频问题 |
| 缺点 | 文件体积大、不适合大量存储和网络传输、元数据能力相对一般 |

#### 文件大小

常见 WAV 是无压缩 PCM，所以文件大小主要由采样率、位深、声道数和时长决定。

```text
文件大小 ≈ 采样率 × 位深 × 声道数 × 时长
```

例如 CD 规格：

```text
44.1kHz × 16bit × 2声道
```

一分钟大约 10MB 左右。

#### 学习意义

WAV 很适合用来理解音频基础概念：

- PCM
- 采样率
- 位深
- 声道
- 无压缩音频
- 播放器输出是否保持原始规格

### FLAC

#### 核心概念

FLAC 全称是 Free Lossless Audio Codec，是一种无损压缩音频格式。

它会把 PCM 音频数据进行无损压缩，播放时再解码还原成 PCM 数据。

```text
PCM 原始数据
    ↓ 无损压缩
FLAC 文件
    ↓ 解码
PCM 数据
```

#### 无损压缩

FLAC 的重点是"无损"。压缩后文件体积会变小，但解码后可以还原出原来的 PCM 音频数据。

这和普通文件压缩类似：压缩是为了减小体积，不是为了丢掉音频信息。

#### 文件信息

一个 FLAC 文件通常可以包含：

| 信息 | 说明 |
| --- | --- |
| 采样率 | 例如 44.1kHz、48kHz、96kHz、192kHz |
| 位深 | 例如 16bit、24bit、32bit |
| 声道数 | 单声道、双声道、多声道 |
| 音频数据 | 无损压缩后的音频数据 |
| 元数据 | 歌名、歌手、专辑、封面、曲目号等 |

#### 常见规格

FLAC 常见规格包括：

- 16bit / 44.1kHz
- 24bit / 48kHz
- 24bit / 96kHz
- 24bit / 192kHz

#### 学习意义

FLAC 是本地无损音乐库里非常常见的格式。播放器播放 FLAC 时通常会先解码成 PCM，再输出到后级设备。

```text
FLAC 文件
    ↓ 播放器解码
PCM 数据
    ↓ 数字输出 / 内部 DAC
DAC / 功放 / 有源音箱
```

### MQA

#### 核心概念

MQA 全称是 Master Quality Authenticated，可以理解为一种高解析音频的压缩、认证和还原方案。

它不是单纯的文件后缀，很多 MQA 音乐会封装在 FLAC 里。因此看到 `.flac` 文件，不代表它一定不是 MQA。

#### 与 FLAC 的关系

FLAC 是一种无损音频封装 / 编码格式，MQA 可以封装在 FLAC 中。

```text
FLAC 不一定是 MQA
MQA 可能是 FLAC 文件
```

播放器可能把 MQA 文件显示为 FLAC，但如果软件或 DAC 支持 MQA，播放时会识别其中的 MQA 信息。

#### 展开

MQA 的核心概念是"展开"，也叫 unfold。可以理解为把编码时折叠进去的高解析信息逐级还原。

```text
MQA 文件 / 流媒体
        ↓
播放器软件识别 MQA
        ↓
第一次展开
        ↓
DAC 继续渲染或完整解码
```

常见情况：

| 类型 | 说明 |
| --- | --- |
| 不展开 | 当普通 FLAC / PCM 播放，有声音，但没有发挥 MQA 能力 |
| 软件第一展开 | 播放器完成第一次展开，输出较高采样率 PCM |
| DAC 渲染 | 播放器完成第一展开，DAC 继续渲染 |
| DAC 完整解码 | DAC 自己完成 MQA 识别、展开和渲染 |

#### 采样率显示

MQA 文件原始显示可能是 44.1kHz 或 48kHz，但展开后可能变成 88.2kHz、96kHz、176.4kHz 或 192kHz。

因此 MQA 不能只看文件显示的采样率，还要看播放器和 DAC 的处理结果。

### DSF / DFF

#### 核心概念

DSF 和 DFF 是 DSD 音频的两种主要文件格式（容器），它们里面装的是 DSD 编码的 1-bit 位流数据，而不是 PCM 数据。

```text
DSD 编码的 1-bit 位流 → DSF 或 DFF 文件
```

这与 WAV 装 PCM 是对等的关系：

```text
PCM 编码数据 → WAV 文件
DSD 编码数据 → DSF / DFF 文件
```

#### DSF 与 DFF 的区别

| 特性 | DSF | DFF |
| --- | --- | --- |
| 全称 | DSD Stream File | DSD Interchange File Format |
| 开发者 | Sony | Philips |
| 元数据支持 | 支持内嵌 ID3 标签（歌名、歌手、封面等） | 元数据能力较弱 |
| 适用场景 | 更常见，适合本地音乐库管理 | 较老的 DSD 文件或 SACD 抓取 |

两种格式在音频数据上本质相同，差异主要在元数据支持和兼容性上。日常使用中 DSF 更友好，因为它能内嵌封面和标签信息。

#### 文件信息

一个 DSF / DFF 文件通常包含：

| 信息 | 说明 |
| --- | --- |
| 采样率 | DSD64 / DSD128 / DSD256 / DSD512 |
| 音频数据 | 1-bit DSD 位流 |
| 元数据 | DSF 可内嵌封面、歌名等（DFF 较弱） |

#### 优点与缺点

| 类型 | 内容 |
| --- | --- |
| 优点 | DSD 原生封装，保留完整 DSD 信息，适合高品质音乐收藏 |
| 缺点 | 文件体积大，播放需要支持 DSD 解码的播放器和 DAC，兼容性不如 PCM 体系 |

#### 学习意义

DSF 和 DFF 是理解 DSD 音频从存储到播放的关键一环：

```text
DSD 编码方式（基础知识）
    ↓
DSF / DFF 文件（存储容器，本章节）
    ↓
播放器 + 输出方式（见输出与解码章节）
    ↓
DSD 兼容 DAC
```

## 输出与解码

### DSD 输出方式

#### 核心概念

播放 DSD 文件时，播放器到 DAC 之间的传输方式有三种常见选择：DoP、D2P、Native。选择哪种取决于播放器、接口和 DAC 各自支持什么。

```text
DSF / DFF 文件
    ↓ 播放器读取
DSD 数据
    ↓ 选择输出方式
DoP / D2P / Native
    ↓
DAC
```

#### 三种方式的对比

| 输出方式 | 全称 / 含义 | 本质 | 特点 | 常见场景 |
| --- | --- | --- | --- | --- |
| DoP | DSD over PCM | 把 DSD 数据封装进 PCM 数据帧里传输 | 传输链路看起来像 PCM，但 DAC 识别后会还原为 DSD | USB、光纤、同轴、部分 HDMI 场景 |
| D2P | DSD to PCM | 把 DSD 转换成 PCM 后输出 | 兼容性最好，但已经不是原生 DSD 输出 | DAC 或接口不支持 DSD 时使用 |
| Native | Native DSD | 直接传输原生 DSD 数据 | 理论上最直接，但需要播放器、驱动、接口和 DAC 都支持 | USB Audio 输出到支持 Native DSD 的 DAC |

#### DoP

DoP 全称是 DSD over PCM。它不是把 DSD 转成 PCM，而是把 DSD 数据封装进 PCM 数据帧中传输。

这样做的好处是可以借用原本支持 PCM 的传输通道，例如 USB、光纤、同轴等。对传输链路来说，它看起来像 PCM；但支持 DoP 的 DAC 识别后，会把里面的 DSD 数据还原出来。

#### D2P

D2P 是 DSD to PCM，也就是把 DSD 转换成 PCM 后输出。

它的兼容性较好，因为很多设备都能接收 PCM。但经过 D2P 后，输出已经不是原生 DSD，而是 PCM。

#### Native

Native 是原生 DSD 输出，播放器直接把 DSD 数据传给 DAC。

它通常要求播放器、输出接口、驱动和 DAC 都支持 Native DSD。常见场景是通过 USB Audio 输出到支持 Native DSD 的外部 DAC。
