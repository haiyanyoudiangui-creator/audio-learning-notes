### 主要功能

#### 播放器 / 数播类重点关注

- 本地音频文件播放。
- 网络音乐 / 串流播放能力。
- 音频解码与输出能力。
- 专辑封面、歌词、元数据显示。
- 多种输出接口下的采样率、位深、声道表现。
- 手机 App、遥控器或网页控制能力。

#### 影音播放器类重点关注

- 本地视频 / 蓝光原盘 / 硬盘播放。
- 4K / 8K 视频输出能力。
- HDMI 音视频输出。
- 音频源码透传、解码、多声道输出。
- 字幕、海报墙、刮削、硬盘管理。

#### 解码器 / 耳放类重点关注

- USB DAC、同轴、光纤、AES、I2S 等输入能力。
- RCA、XLR、耳机口等输出能力。
- PCM / DSD 支持能力。
- 推力、增益、滤波器、音量控制。

#### 功放类重点关注

- 输入接口。
- 输出功率。
- 阻抗适配。
- 声道配置。
- 与数播 / 解码器的搭配方式。

### 数播产品分类逻辑

从工作和学习角度看，开博尔数播产品可以先不要按型号名复杂分类，而是按“是否内置解码板、是否具备模拟音频输出”分成两大类。

```text
开博尔数播产品
├── 纯数播
│   └── 重点：播放系统、音乐库、串流协议、数字音频输出
└── 带解码板的数播
    └── 重点：播放系统、数字音频输出、内部 DAC、模拟音频输出
```

| 类型 | 核心区别 | 常见输出 | 学习重点 |
| --- | --- | --- | --- |
| 纯数播 | 主要负责播放、管理、串流和数字输出，不负责最终模拟解码 | USB Audio、HDMI、光纤、同轴、I2S 等 | 播放链路、数字输出、协议、采样率、SRC、USB 独占 |
| 带解码板的数播 | 内置 DAC / 解码板，可直接输出模拟音频 | RCA、XLR、耳机口，同时可能保留数字输出 | DAC 芯片、模拟输出、音量控制、电源、底噪、失真 |

#### 纯数播信号链

```text
音乐文件 / 流媒体
        ↓
播放器系统
        ↓
解码前或处理后的数字音频
        ↓
数字输出：USB / HDMI / 光纤 / 同轴 / I2S
        ↓
外部 DAC / AV 功放 / 解码器
        ↓
功放 / 有源音箱 / 耳机系统
```

纯数播的排查重点通常在：

- 音乐文件或流媒体是否正常。
- 播放器系统是否正确识别格式、采样率、位深。
- 是否经过 SRC。
- 输出接口是否选择正确。
- 外部 DAC 或功放是否正确接收数字信号。

#### 带解码板数播信号链

```text
音乐文件 / 流媒体
        ↓
播放器系统
        ↓
数字音频处理
        ↓
内部 DAC / 解码板
        ↓
模拟输出：RCA / XLR / 耳机口
        ↓
功放 / 有源音箱 / 耳机
```

带解码板数播的排查重点通常在：

- 播放链路是否正常。
- 数字音频是否正确进入内部 DAC。
- DAC 支持的 PCM / DSD / MQA 能力。
- 模拟输出接口是否正常。
- 音量、增益、静音、平衡输出是否设置正确。
- 后级功放、有源音箱或耳机是否匹配。

## 音频文件格式

### WAV

#### 核心概念

WAV 全称通常写作 Waveform Audio File Format，是一种常见的音频容器格式。

它本身更像一个“文件壳”，里面最常见装的是 PCM 音频数据。

```text
WAV 通常 = PCM 原始音频数据 + 文件头信息
```

#### 与 PCM 的关系

PCM 是音频数据的编码方式，WAV 是文件格式。

```text
PCM = 音频数据怎么存
WAV = 把 PCM 数据装进一个文件里
```

所以常见的 `16bit / 44.1kHz WAV`、`24bit / 96kHz WAV`，实际描述的是 WAV 文件里面 PCM 数据的规格。

#### 文件信息

一个 WAV 文件通常会记录：

| 信息 | 说明 |
| --- | --- |
| 采样率 | 例如 44.1kHz、48kHz、96kHz、192kHz |
| 位深 | 例如 16bit、24bit、32bit |
| 声道数 | 单声道、双声道、多声道 |
| 编码格式 | 最常见是 PCM，也可能是 ADPCM、浮点 PCM 等 |
| 音频数据 | 真正的声音采样数据 |

#### 优点与缺点

| 类型 | 内容 |
| --- | --- |
| 优点 | 结构简单、兼容性强、通常无压缩、适合分析音频问题 |
| 缺点 | 文件体积大、不适合大量存储和网络传输、元数据能力相对一般 |

#### 文件大小

常见 WAV 是无压缩 PCM，所以文件大小主要由采样率、位深、声道数和时长决定。

```text
文件大小 ≈ 采样率 × 位深 × 声道数 × 时长
```

例如 CD 规格：

```text
44.1kHz × 16bit × 2声道
```

一分钟大约 10MB 左右。

#### 学习意义

WAV 很适合用来理解音频基础概念：

- PCM
- 采样率
- 位深
- 声道
- 无压缩音频
- 播放器输出是否保持原始规格

### FLAC

#### 核心概念

FLAC 全称是 Free Lossless Audio Codec，是一种无损压缩音频格式。

它会把 PCM 音频数据进行无损压缩，播放时再解码还原成 PCM 数据。

```text
PCM 原始数据
    ↓ 无损压缩
FLAC 文件
    ↓ 解码
PCM 数据
```

#### 无损压缩

FLAC 的重点是“无损”。压缩后文件体积会变小，但解码后可以还原出原来的 PCM 音频数据。

这和普通文件压缩类似：压缩是为了减小体积，不是为了丢掉音频信息。

#### 文件信息

一个 FLAC 文件通常可以包含：

| 信息 | 说明 |
| --- | --- |
| 采样率 | 例如 44.1kHz、48kHz、96kHz、192kHz |
| 位深 | 例如 16bit、24bit、32bit |
| 声道数 | 单声道、双声道、多声道 |
| 音频数据 | 无损压缩后的音频数据 |
| 元数据 | 歌名、歌手、专辑、封面、曲目号等 |

#### 常见规格

FLAC 常见规格包括：

- 16bit / 44.1kHz
- 24bit / 48kHz
- 24bit / 96kHz
- 24bit / 192kHz

#### 学习意义

FLAC 是本地无损音乐库里非常常见的格式。对数播产品来说，播放 FLAC 时通常会先解码成 PCM，再输出到后级设备。

```text
FLAC 文件
    ↓ 播放器解码
PCM 数据
    ↓ 数字输出 / 内部 DAC
DAC / 功放 / 有源音箱
```

### DSD

#### 核心概念

DSD 是一种不同于 PCM 的数字音频编码方式，常见于 SACD、DSF、DFF、ISO-DSD 等文件。

学习 DSD 时，除了看文件本身是 DSD64、DSD128、DSD256 还是 DSD512，还要理解播放器和 DAC 之间采用哪种输出方式。

#### 三种输出方式

常见 DSD 输出方式有三种：DoP、D2P、Native。

| 输出方式 | 全称 / 含义 | 本质 | 特点 | 常见场景 |
| --- | --- | --- | --- | --- |
| DoP | DSD over PCM | 把 DSD 数据封装进 PCM 数据帧里传输 | 传输链路看起来像 PCM，但 DAC 识别后会还原为 DSD | USB、光纤、同轴、部分 HDMI 场景 |
| D2P | DSD to PCM | 把 DSD 转换成 PCM 后输出 | 兼容性最好，但已经不是原生 DSD 输出 | DAC 或接口不支持 DSD 时使用 |
| Native | Native DSD | 直接传输原生 DSD 数据 | 理论上最直接，但需要播放器、驱动、接口和 DAC 都支持 | USB Audio 输出到支持 Native DSD 的 DAC |

#### DoP

DoP 全称是 DSD over PCM。它不是把 DSD 转成 PCM，而是把 DSD 数据封装进 PCM 数据帧中传输。

这样做的好处是可以借用原本支持 PCM 的传输通道，例如 USB、光纤、同轴等。对传输链路来说，它看起来像 PCM；但支持 DoP 的 DAC 识别后，会把里面的 DSD 数据还原出来。

#### D2P

D2P 是 DSD to PCM，也就是把 DSD 转换成 PCM 后输出。

它的兼容性较好，因为很多设备都能接收 PCM。但经过 D2P 后，输出已经不是原生 DSD，而是 PCM。

#### Native

Native 是原生 DSD 输出，播放器直接把 DSD 数据传给 DAC。

它通常要求播放器、输出接口、驱动和 DAC 都支持 Native DSD。常见场景是通过 USB Audio 输出到支持 Native DSD 的外部 DAC。

### MQA

#### 核心概念

MQA 全称是 Master Quality Authenticated，可以理解为一种高解析音频的压缩、认证和还原方案。

它不是单纯的文件后缀，很多 MQA 音乐会封装在 FLAC 里。因此看到 `.flac` 文件，不代表它一定不是 MQA。

#### 与 FLAC 的关系

FLAC 是一种无损音频封装 / 编码格式，MQA 可以封装在 FLAC 中。

```text
FLAC 不一定是 MQA
MQA 可能是 FLAC 文件
```

播放器可能把 MQA 文件显示为 FLAC，但如果软件或 DAC 支持 MQA，播放时会识别其中的 MQA 信息。

#### 展开

MQA 的核心概念是“展开”，也叫 unfold。可以理解为把编码时折叠进去的高解析信息逐级还原。

```text
MQA 文件 / 流媒体
        ↓
播放器软件识别 MQA
        ↓
第一次展开
        ↓
DAC 继续渲染或完整解码
```

常见情况：

| 类型 | 说明 |
| --- | --- |
| 不展开 | 当普通 FLAC / PCM 播放，有声音，但没有发挥 MQA 能力 |
| 软件第一展开 | 播放器完成第一次展开，输出较高采样率 PCM |
| DAC 渲染 | 播放器完成第一展开，DAC 继续渲染 |
| DAC 完整解码 | DAC 自己完成 MQA 识别、展开和渲染 |

#### 采样率显示

MQA 文件原始显示可能是 44.1kHz 或 48kHz，但展开后可能变成 88.2kHz、96kHz、176.4kHz 或 192kHz。

因此 MQA 不能只看文件显示的采样率，还要看播放器和 DAC 的处理结果。
